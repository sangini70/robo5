0. 문서의 목적

이 문서는 robo5 프로젝트의 공통 디자인 헌법이다.

이 문서는 특정 화면 한 장을 만들기 위한 일회성 프롬프트가 아니다.

다음 도구와 작업자가 공개 사이트의 디자인을 설계하거나 구현할 때 공통 기준으로 사용한다.

1. 최상위 디자인 명제

좋은 출판사의 경제 교양서를 온라인으로 읽는 느낌.

robo5의 공개 사이트는 뉴스 속보를 빠르게 소비하는 화면이 아니다.

투자 수익률을 강조하는 금융상품 판매 화면도 아니다.

사용자는 차분하게 개념을 이해하고, 관련 지식을 연결하며, 다음에 무엇을 읽어야 하는지 자연스럽게 발견해야 한다.

디자인은 콘텐츠를 장식하는 역할보다 다음 역할을 우선한다.

이해를 돕는다.

정보의 위계를 보여준다.

현재 위치를 알려준다.

다음 학습 경로를 제안한다.

긴 글을 편안하게 읽게 한다.

사이트에 대한 신뢰를 형성한다.

3. 사용자에게 남겨야 할 인상

사용자가 첫 화면을 본 뒤 다음과 같이 느껴야 한다.

금융이 생각보다 어렵지 않다.

내용이 체계적으로 정리되어 있다.

광고용 블로그가 아니라 믿고 읽을 수 있는 지식 플랫폼이다.

지금 무엇을 읽어야 하는지 알 수 있다.

한 글을 읽고 나니 다음 글도 궁금하다.

나중에 다시 찾아오고 싶다.

사용자 감정의 흐름은 다음과 같다.

낯설음
→ 안도감
→ 이해
→ 신뢰
→ 호기심
→ 탐색
→ 재방문

디자인은 불안, 조급함, 과도한 기대, 투자 충동을 유발하지 않는다.

4. 사용자 경험 철학

4.1 사용자는 길을 잃으면 안 된다

모든 주요 화면은 다음 질문에 답해야 한다.

나는 지금 무엇을 보고 있는가?

이 페이지에서 무엇을 배우는가?

이 내용은 전체 지식 구조에서 어디에 속하는가?

먼저 읽어야 할 내용은 무엇인가?

다음에 읽을 내용은 무엇인가?

관련 개념은 무엇인가?

4.2 화면은 학습의 방향을 보여줘야 한다

robo5의 기본 경험은 다음과 같다.

발견
→ 이해
→ 관계 파악
→ 비교
→ 다음 글
→ 학습 확장

홈페이지는 최신 글을 나열하는 장소가 아니라 학습의 출발점이다.

카테고리는 전체 분야를 탐색하는 장소다.

허브는 하나의 주제를 순서대로 학습하는 장소다.

글 상세는 한 질문에 답하고 다음 지식으로 연결하는 장소다.

검색은 단순 결과 목록이 아니라 지식 구조로 다시 진입하는 장소다.

4.3 사용자 행동을 강요하지 않는다

CTA는 최소화한다.

팝업으로 탐색을 방해하지 않는다.

자동 슬라이더로 사용자의 속도를 빼앗지 않는다.

과도한 고정 요소를 사용하지 않는다.

클릭을 유도하기 위해 불안이나 조급함을 만들지 않는다.

5. 콘텐츠 철학과 정보 구조

5.1 콘텐츠 흐름

디자인은 다음 학습 구조를 시각적으로 지원한다.

정보
→ 이해
→ 비교
→ 관계
→ 다음 글
→ 학습

또는 프로젝트 Flow 기준에 따라 다음을 지원한다.

정의
→ 원인
→ 방법
→ 비교
→ 판단
→ 실행

페이지 유형과 주제에 따라 표현 방식은 달라질 수 있지만, 사용자가 현재 단계와 다음 단계를 알 수 있어야 한다.

5.2 Knowledge 구조 보호

디자인은 다음 관계를 감추지 않는다.

Category
→ Hub
→ Topic
→ Knowledge Node
→ Previous / Next
→ Related Node

카테고리와 허브를 같은 개념처럼 취급하지 않는다.

카테고리

여러 허브와 글을 탐색하는 분류 화면

핵심 허브, 최신 글, 세부 Topic, 관련 분야를 제공

넓은 탐색을 지원

지식 허브

하나의 주제를 단계별로 학습하는 화면

현재 단계, 이전 Node, 다음 Node, 관련 Node를 제공

깊은 학습을 지원

현재 구현상 두 기능이 한 화면에 함께 존재해야 한다면 다음처럼 구분한다.

상단: 허브형 학습 안내
하단: 카테고리형 콘텐츠 탐색

5.3 실제 콘텐츠 우선

디자인 시안과 구현에서는 가능한 한 실제 콘텐츠 제목을 사용한다.

임의의 영어 금융 문구, 가짜 수익률, 가짜 뉴스, 가짜 이용자 수를 넣지 않는다.

대표 콘텐츠 예시:

환율이란 무엇인가

환율은 어떻게 결정될까

환율 보는 법

환율과 금리의 관계

원달러 환율이 중요한 이유

환율이 물가에 미치는 영향

고정환율과 변동환율 차이

로보어드바이저 수수료란 무엇인가

AI는 금융을 어떻게 바꾸고 있을까

금융용어 예시:

환율전가효과

기준금리

실효환율

생산자물가지수

소비자물가지수

8. 다국어와 Global Edition 원칙

8.1 기본 구조

한국어와 영어는 동일한 디자인 시스템을 공유한다.

그러나 영어 페이지는 한국어 페이지의 단순 번역본이 아니다.

한국어 홈페이지
/

Global Edition
/en

영어권 사용자의 검색 의도와 문화적 맥락에 따라 콘텐츠 우선순위는 달라질 수 있다.

8.2 언어 전환

한국어, English 또는 Global 표기를 명확하게 사용한다.

국기만으로 언어를 표시하지 않는다.

국기를 보조 아이콘으로 사용할 수 있으나 텍스트 라벨을 함께 둔다.

언어 전환은 단순 글 필터가 아니라 각 언어의 홈페이지로 이동한다.

한국어는 /, 영어는 /en을 기본 진입점으로 고려한다.

8.3 레이아웃 확장성

컴포넌트는 다음 차이를 견뎌야 한다.

대소문자 사용

줄바꿈 차이

9. Design Tokens

디자인 토큰은 구현 도구와 관계없이 일관되게 유지한다.

정확한 값은 실제 구현 과정에서 조정할 수 있으나 아래 방향을 벗어나지 않는다.

용도:

9.3 Spacing

여백은 장식이 아니라 정보의 관계를 보여주는 도구다.

같은 의미의 요소는 가깝게 둔다.

다른 논리 단위는 충분히 분리한다.

섹션 간 간격은 카드 내부 간격보다 명확히 크게 한다.

모바일에서도 문단과 섹션 사이의 호흡을 유지한다.

일관된 4px 또는 8px 기반 spacing scale을 사용한다.

9.4 Grid

전체 대응 폭: 최대 1920px

공통 콘텐츠 폭: 약 1280~1360px

일부 와이드 섹션: 최대 1440px

글 본문: 약 720~780px

데스크톱 좌우 여백: 유동형

모바일 최소 기준: 360px

대표 모바일 시안 기준: 390px

1440px 전체를 항상 콘텐츠로 채우지 않는다.

본문, 목록, 사이드바가 각자의 읽기 목적에 맞는 폭을 가져야 한다.

9.5 Radius

둥근 모서리는 절제해서 사용한다.

모든 요소에 radius를 적용하지 않는다.

카드, 입력창, 버튼은 역할에 따라 다른 radius를 사용할 수 있다.

과도하게 둥근 SaaS형 pill UI를 피한다.

20. 최종 선언

robo5의 디자인은 금융을 화려하게 보이게 만드는 것이 아니다.

복잡한 금융 지식을 이해할 수 있게 만드는 것이다.

디자인은 사용자를 붙잡기보다 길을 보여준다.

콘텐츠를 압도하지 않고, 구조를 드러내며, 다음 학습으로 연결한다.

모든 화면은 다음 문장을 기준으로 판단한다.

좋은 출판사의 경제 교양서를 온라인으로 읽는 느낌.
