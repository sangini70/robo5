# MY_GSTACK_APPLIED.md

## 목적

이 문서는 robo5 프로젝트에서
my_gstack 운영체계 중
실제로 적용하는 철학과 skill 범위를 정의한다.

목표:

- robo5 방향 일관성 유지
- AI 작업 품질 안정화
- production 중심 운영 유지
- 탐험형 UX 유지
- 과설계 방지
- SEO 엔진 방향 유지

---

# 1. 적용 기본 원칙

my_gstack 전체를 무조건 적용하지 않는다.

robo5 목적에 맞는 부분만 선택적으로 적용한다.

robo5는:

- 금융 탐험 플랫폼
- 콘텐츠 미디어
- SEO 엔진
- 체류형 UX

중심 프로젝트다.

따라서:

- 복잡한 SaaS 운영 구조
- 과도한 시스템 abstraction
- enterprise-style architecture

는 우선순위가 아니다.

# 3. 현재 적용하는 skill

## browser-operations

적용 목적:

- production UX 검증
- 실제 브라우저 흐름 확인
- 모바일 흐름 확인
- hydration/console 오류 확인

핵심:

localhost보다
실제 production UX 우선.

---

## frontend-operations

적용 목적:

- 최소 수정 유지
- 구조 안정성 유지
- incremental improvement 유지

핵심:

불필요한 abstraction 금지.

---

## BLOG_UX_DESIGN

적용 목적:

- 탐험형 콘텐츠 흐름 유지
- story-like navigation 유지
- related exploration 강화

핵심:

사용자는
“대시보드 사용”
이 아니라

“탐험”
하고 있어야 한다.

---

## production-verification

적용 목적:

- 실제 production 검증
- 모바일 UX 확인
- scroll rhythm 확인
- SEO rendering 확인

핵심:

build 성공보다
실제 UX가 중요하다.

---

# 4. robo5에서 중요하게 적용하는 철학

robo5는:

- AI스럽지 않은 UX
- 인간적인 흐름
- 탐험 구조
- 읽기 경험
- 체류 리듬
- 자연스러운 콘텐츠 연결

을 가장 중요하게 본다.

따라서:

- 기능 폭주
- 과도한 animation
- SaaS dashboard 느낌
- 과설계
- AI 대량생성 느낌

을 경계한다.

---

# 5. robo5에서 현재 우선순위가 아닌 것

현재 robo5는 아래를 우선하지 않는다.

- enterprise architecture
- microservice 분리
- 복잡한 state machine
- advanced abstraction
- heavy optimization
- 과도한 automation
- 복잡한 CMS 구조

현재 우선순위:

- UX 흐름
- SEO 구조
- 콘텐츠 연결
- 체류시간
- 탐험 리듬
- production 품질

---

# 6. 적용 방식

my_gstack는:

“강제 규칙”

이 아니라

“운영 reference”

로 사용한다.

중요:

robo5 실제 목적과 충돌할 경우
robo5 방향을 우선한다.

---

# 7. 최종 방향

robo5는:

- 탐험형 금융 미디어
- AI 시대 금융 교양 플랫폼
- SEO 기반 콘텐츠 엔진
- 체류형 UX 플랫폼

으로 발전한다.

my_gstack 적용 목적도:

복잡한 시스템 구축이 아니라

- 흐름 강화
- UX 안정화
- production 품질 향상
- 탐험 경험 강화

에 있다.
